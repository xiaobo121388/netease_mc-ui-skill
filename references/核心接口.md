# UI 接口 (API) 参考

本指南列出了网易我的世界 UI 开发中最常用的 Python API。这些 API 主要通过 `clientApi` 或 `ScreenNode` 实例调用。

## 1. 客户端系统 API (ClientSystem)

这些 API 通常在你的自定义 `ClientSystem` 类中调用。

### 注册与创建 UI

- **`RegisterUI(namespace, uiName, clsPath, screenDef)`**
  - **描述**：注册一个自定义 UI。
  - **参数**：
    - `namespace` (str): UI 的命名空间。
    - `uiName` (str): UI 的名称。
    - `clsPath` (str): 对应的 Python 类的路径（如 `"my_mod.client.ui.my_ui.MyUI"`）。
    - `screenDef` (str): 对应的 JSON 界面定义（如 `"my_ui.main"`）。

- **`CreateUI(namespace, uiName, param)`**
  - **描述**：创建并显示一个已注册的 UI。
  - **参数**：
    - `namespace` (str): UI 的命名空间。
    - `uiName` (str): UI 的名称。
    - `param` (dict): 传递给 UI 类的参数字典（如 `{"isHud": 1}`）。
  - **返回**：创建的 `ScreenNode` 实例。

- **`GetUI(namespace, uiName)`**
  - **描述**：获取一个已创建的 UI 实例。
  - **返回**：`ScreenNode` 实例，如果未创建则返回 `None`。

### 堆栈管理

- **`PushScreen(namespace, uiName, param)`**
  - **描述**：将一个 UI 推入屏幕堆栈并显示。常用于全屏界面（如菜单、背包）。
  - **注意**：使用此方法创建的 UI 会自动加载其所需的贴图资源，适合包含大量贴图的界面（如轮盘）。

- **`PopScreen()`**
  - **描述**：弹出当前位于堆栈顶部的 UI。

## 2. 屏幕节点 API (ScreenNode)

这些 API 在你的自定义 UI 类（继承自 `ScreenNode`）中调用，用于操作 UI 控件。

### 控件获取与基础操作

- **`GetBaseUIControl(path)`**
  - **描述**：获取指定路径的控件实例。
  - **参数**：`path` (str) 控件的绝对路径（如 `"/panel/my_button"`）。
  - **返回**：`BaseUIControl` 实例。

- **`GetAllChildrenPath(path)`**
  - **描述**：获取指定节点下的所有子节点路径。常用于调试和查找原生 UI 的控件路径。
