# UI 接口 (API) 参考

本指南列出了网易我的世界 UI 开发中最常用的 Python API。这些 API 主要通过 `clientApi` 或 `ScreenNode` 实例调用。

## 1. 客户端系统 API (ClientSystem)

这些 API 通常在你的自定义 `ClientSystem` 类中调用。

### 注册与创建 UI

- **`RegisterUI(namespace, uiName, clsPath, screenDef)`**
  - **描述**：注册一个自定义 UI。
  - **参数**：
    - `namespace` (str): UI 的命名空间。
    - `uiName` (str): UI 的名称。
    - `clsPath` (str): 对应的 Python 类的路径（如 `"my_mod.client.ui.my_ui.MyUI"`）。
    - `screenDef` (str): 对应的 JSON 界面定义（如 `"my_ui.main"`）。

- **`CreateUI(namespace, uiName, param)`**
  - **描述**：创建并显示一个已注册的 UI。
  - **参数**：
    - `namespace` (str): UI 的命名空间。
    - `uiName` (str): UI 的名称。
    - `param` (dict): 传递给 UI 类的参数字典（如 `{"isHud": 1}`）。
  - **返回**：创建的 `ScreenNode` 实例。

- **`GetUI(namespace, uiName)`**
  - **描述**：获取一个已创建的 UI 实例。
  - **返回**：`ScreenNode` 实例，如果未创建则返回 `None`。

### 堆栈管理

- **`PushScreen(namespace, uiName, param)`**
  - **描述**：将一个 UI 推入屏幕堆栈并显示。常用于全屏界面（如菜单、背包）。
  - **注意**：使用此方法创建的 UI 会自动加载其所需的贴图资源，适合包含大量贴图的界面（如轮盘）。

- **`PopScreen()`**
  - **描述**：弹出当前位于堆栈顶部的 UI。

## 2. 屏幕节点 API (ScreenNode)

这些 API 在你的自定义 UI 类（继承自 `ScreenNode`）中调用，用于操作 UI 控件。

### 控件获取与基础操作

- **`GetBaseUIControl(path)`**
  - **描述**：获取指定路径的控件实例。
  - **参数**：`path` (str) 控件的绝对路径（如 `"/panel/my_button"`）。
  - **返回**：`BaseUIControl` 实例。

- **`GetAllChildrenPath(path)`**
  - **描述**：获取指定节点下的所有子节点路径。常用于调试和查找原生 UI 的控件路径。

- **`SetVisible(path, visible)`**
  - **描述**：设置控件的可见性。
  - **参数**：`visible` (bool) 是否可见。

- **`SetPosition(path, pos)`** / **`SetSize(path, size)`**
  - **描述**：设置控件的位置或大小。
  - **参数**：`pos` / `size` (tuple) 包含两个元素的元组，如 `(100, 50)`。

### 事件绑定

- **`AddTouchEventHandler(path, callback, args)`**
  - **描述**：为控件动态绑定触摸/点击事件。
  - **参数**：
    - `path` (str): 控件路径。
    - `callback` (function): 回调函数。
    - `args` (dict): 额外参数，如 `{"isSwallow": True}` 表示吞噬事件。

### 特殊控件操作

- **`SetUiItem(path, itemName, auxValue, isEnchanted)`**
  - **描述**：为 `ItemRenderer` 控件设置要显示的物品。
  - **参数**：
    - `itemName` (str): 物品标识符（如 `"minecraft:apple"`）。
    - `auxValue` (int): 附加值。
    - `isEnchanted` (bool): 是否显示附魔光效。

- **`RenderPaperDoll(path, param)`**
  - **描述**：为 `NeteasePaperDoll` 控件设置要渲染的模型或实体。
  - **参数**：`param` (dict) 包含渲染参数的字典，如 `{"entity_id": "123", "scale": 1.0}`。

- **`SetEditTextMaxLength(path, maxLength)`**
  - **描述**：动态设置 `TextEditBox` 控件的最大输入长度。

- **`SetText(path, text)`**
  - **描述**：设置 `Label` 控件的文本内容。建议优先使用数据绑定。

## 3. 控件实例 API (BaseUIControl)

通过 `GetBaseUIControl` 获取到的实例，可以直接调用其方法。

- **`GetVisible()`**：获取当前可见性。
- **`GetPosition()`**：获取当前位置。
- **`GetSize()`**：获取当前大小。
- **`SetAlpha(alpha)`**：设置透明度（0.0 - 1.0）。

*注意：在大多数情况下，直接通过 `ScreenNode` 的方法（如 `self.SetVisible(path, True)`）操作控件比先获取实例再操作更方便。获取实例通常用于需要频繁读取控件状态的场景。*
