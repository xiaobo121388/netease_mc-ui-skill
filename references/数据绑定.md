# UI 数据绑定与按钮映射

数据绑定是网易我的世界 UI 系统中连接 JSON 视图和 Python 逻辑的核心机制。通过数据绑定，可以实现 UI 属性的自动更新和用户交互事件的响应。

## 1. 数据绑定基础

数据绑定分为两部分：
1. **JSON 声明**：在控件的 `bindings` 数组中声明绑定关系。
2. **Python 绑定**：在 Python 脚本中使用 `@ViewBinder.binding` 装饰器绑定对应的函数。

### 绑定类型 (Bind Flag)

`ViewBinder` 提供了多种绑定类型，用于不同类型的数据和事件：

| 绑定类型 | 绑定方式 | 解释 |
| :--- | :--- | :--- |
| `BF_ButtonClickUp` | binding | 绑定按钮的 Up 事件 |
| `BF_ButtonClickDown` | binding | 绑定按钮的 Down 事件 |
| `BF_ButtonClick` | binding | 同时绑定 Up 和 Down 事件 |
| `BF_ButtonClickCancel` | binding | 绑定按钮的 Cancel 事件 |
| `BF_InteractButtonClick` | binding | 绑定游戏原生的按钮点击事件 |
| `BF_BindBool` | binding / binding_collection | 绑定 Bool 变量（如可见性） |
| `BF_BindInt` | binding / binding_collection | 绑定 Int 变量 |
| `BF_BindFloat` | binding / binding_collection | 绑定 Float 变量（如进度条） |
| `BF_BindString` | binding / binding_collection | 绑定 String 变量（如文本内容） |
| `BF_BindGridSize` | binding | 绑定 GridSize 变量 |
| `BF_BindColor` | binding / binding_collection | 绑定颜色变量 |
| `BF_EditChanged` | binding | 绑定输入框输入改变事件 |
| `BF_EditFinished` | binding | 绑定输入框输入完成事件 |
| `BF_ToggleChanged` | binding | 开关状态改变事件 |

### 返回值 (ViewRequest)

绑定的 Python 函数通常需要返回一个 `ViewRequest` 常量，告诉引擎接下来如何处理：

- `ViewRequest.Refresh`：刷新 UI，通常在数据改变后返回。
- `ViewRequest.Nothing`：不执行任何操作。
- `ViewRequest.Exit`：退出当前 UI。

## 2. 单个绑定 (View Binding)

单个绑定用于将一个 UI 属性或事件绑定到一个 Python 函数。

### JSON 声明

在 JSON 中，使用 `bindings` 数组声明绑定。`binding_name` 是我们在 Python 中绑定的名称，`binding_name_override` 是引擎内部的变量名（如 `#visible`）。
`binding_name`有两种定义方法：
1. `%命名空间.方法名`, 命名空间需screenNode、python文件名和jsonui的namespace一致。
2. `#控件路径` 直接绑定到控件。

```json
"bindings": [
  {
    "binding_condition": "always",
    "binding_name": "#my_ui.is_visible",
    "binding_name_override": "#visible"
  }
]
```

### Python 绑定

在 Python 中，使用 `@ViewBinder.binding` 装饰器。

```python
@ViewBinder.binding(ViewBinder.BF_BindBool, "#my_ui.is_visible")
def ReturnVisible(self):
    return self.isVisible
```

## 3. 集合绑定 (Collection Binding)

集合绑定适用于列表容器，如 `Grid` 或 `StackPanel`。它允许你为集合中的每个元素绑定数据。

### JSON 声明

首先，在容器控件（如 Grid）中声明 `collection_name`：

```json
"collection_name": "my_grid_collection"
```

然后，在子控件的模板中声明集合绑定。`binding_type` 必须为 `collection`。

```json
"bindings": [
  {
    "binding_collection_name": "my_grid_collection",
    "binding_condition": "always",
    "binding_name": "#my_item.text",
    "binding_type": "collection"
  }
],
"text": "#my_item.text"
```

### Python 绑定

在 Python 中，使用 `@ViewBinder.binding_collection` 装饰器。绑定的函数会接收一个 `index` 参数，表示当前是集合中的第几个元素。

```python
@ViewBinder.binding_collection(ViewBinder.BF_BindString, "my_grid_collection", "#my_item.text")
def OnRefreshItemText(self, index):
    if index < len(self.itemList):
        return self.itemList[index]
    return ""
```

## 4. 按钮映射 (Button Mappings)

按钮映射（Button Mappings）是一种特殊的绑定机制，主要用于将物理按键（如键盘、手柄）或特定的 UI 交互映射到控件的事件上。它常用于按钮、输入面板（InputPanel）等控件。

### 基础用法

在控件的 JSON 定义中，添加 `button_mappings` 数组。每个映射包含三个主要属性：

- `from_button_id`: 触发事件的来源按钮 ID（如 `"button.menu_select"` 代表鼠标左键或手柄确认键）。
- `to_button_id`: 映射到的目标按钮 ID，通常是一个自定义的变量名（如 `"#my_button.click"`）。
- `mapping_type`: 映射的类型，通常为 `"pressed"` 或 `"global"`。

### 示例：InputPanel 拖拽映射

对于 `InputPanel` 控件，要开启拖拽功能，必须配置 `button_mappings` 以接受屏幕点击事件：

```json
"InputPanel": {
  "type": "input_panel",
  "draggable": "both",
  "button_mappings": [
    {
      "from_button_id": "button.menu_select",
      "to_button_id": "#netease_to_button_id",
      "mapping_type": "pressed"
    }
  ]
}
```

### 示例：按钮点击映射
除了通过`SetButtonTouchUpCallback`等接口在Python中绑定按钮回调外，还可以直接在JSON中使用`button_mappings`将按钮点击映射到Python函数：

对于自定义按钮，可以使用 `button_mappings` 将点击事件映射到 Python 脚本中：

```json
"my_button": {
  "type": "button",
  "button_mappings": [
    {
      "from_button_id": "button.menu_select",
      "to_button_id": "#my_button.click",
      "mapping_type": "pressed"
    }
  ]
}
```

在 Python 中，你可以监听这个 `to_button_id`：

```python
@ViewBinder.binding(ViewBinder.BF_ButtonClick, "#my_button.click")
def OnMyButtonClick(self, args):
    print("Button clicked!")
    return ViewRequest.Refresh
```

*注意：大多数情况下，使用 `common.button` 模板时，底层已经配置好了 `button_mappings`，你只需要设置 `$pressed_button_name` 变量即可。*
